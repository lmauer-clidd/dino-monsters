# Nova Forge Studio — Pipeline de Production

## Phase 1 : Pre-Production

### Etape 1.1 — Brainstorm
- Tous les agents participent
- KAELEN pose le cadre, chacun propose
- Objectif : trouver LE concept qui excite tout le monde
- Livrable : **Pitch en 1 phrase**

### Etape 1.2 — Documentation
Equipes en parallele :
- **Game Design** (YUKI + MARCUS + SANA) → GCD.md + GDD.md
- **Art** (ZEPHYR + NOVA + ATLAS + PIXEL) → ART_DIRECTION.md
- **Narrative** (ORION + LYRA) → NARRATIVE_BIBLE.md + WORLD_BIBLE.md
- **Tech** (CIPHER + RUNE) → TECH_ARCHITECTURE.md
- **Business** (VESPER + ECHO) → BUSINESS_MODEL.md + MARKETING_STRATEGY.md

### Livrables Phase 1
- [ ] Game Concept Document (GCD.md)
- [ ] Game Design Document (GDD.md)
- [ ] Art Direction Document (ART_DIRECTION.md)
- [ ] Narrative Bible (NARRATIVE_BIBLE.md)
- [ ] World Bible (WORLD_BIBLE.md)
- [ ] Tech Architecture (TECH_ARCHITECTURE.md)
- [ ] Business Model (BUSINESS_MODEL.md)
- [ ] Marketing Strategy (MARKETING_STRATEGY.md)

## Phase 2 : Prototypage

### Etape 2.1 — Core Mechanics
- SPARK + RUNE : prototype jouable du core loop
- YUKI : validation du game feel (regle des 30 secondes)
- Objectif : "est-ce que c'est fun en 30 secondes ?"

### Etape 2.2 — Art & Audio
- PIXEL : UI/HUD prototype
- HARMONY : sons et musique procedurale
- ZEPHYR : validation du style visuel

### Etape 2.3 — Integration
- CIPHER : integration de tous les systemes
- Test de la boucle complete
- Livrable : **Prototype jouable**

### Livrables Phase 2
- [ ] Prototype jouable (HTML5 / Godot / Unity)
- [ ] Core mechanics validees
- [ ] Art style proof of concept
- [ ] Audio integre

## Phase 3 : Production

### Contenu
- Tous les niveaux/biomes
- Tous les personnages/creatures
- Tous les systemes de jeu
- Narration integree
- UI complete

### Livrables Phase 3
- [ ] Jeu complet (tous les niveaux)
- [ ] UI/UX finalisee
- [ ] Narration integree
- [ ] Sauvegarde fonctionnelle
- [ ] Difficulte equilibree

## Phase 4 : Polish & QA

### Tests
- QA fonctionnel (bugs)
- QA de performance (FPS, memoire)
- QA d'accessibilite
- Playtests externes

### Polish
- Ecran titre anime
- Transitions
- Juice et game feel
- Son et musique finalises

### Livrables Phase 4
- [ ] Zero bug bloquant
- [ ] Performance validee
- [ ] Accessibilite validee
- [ ] Plan de lancement

## Phase 5 : Lancement

- Marketing pre-lancement
- Page store (Steam/itch.io)
- Demo jouable
- Relations createurs de contenu
- Launch day
- Post-launch support

## Regles de validation

- Chaque phase se termine par une **revue** avec KAELEN
- L'utilisateur valide avant de passer a la phase suivante
- Les agents sauvegardent leurs decisions en memoire
- Les retrospectives identifient ce qui a marche et ce qui doit changer
